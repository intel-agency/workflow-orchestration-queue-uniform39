# workflow-orchestration-queue

**Repository:** `intel-agency/workflow-orchestration-queue-uniform39`  
**Type:** Python microservices platform — headless agentic AI orchestration  
**Purpose:** Transforms GitHub Issues into autonomous AI execution orders. A product manager applies a label to an issue; the system claims it, spawns an isolated AI worker in a DevContainer, generates code, runs tests, and submits a Pull Request — without human intervention.

**4-Pillar Architecture:**
1. **The Ear** (`src/notifier/notifier_service.py`) — FastAPI webhook receiver; validates HMAC SHA-256 signatures; applies `agent:queued` label
2. **The State** (GitHub Issues + Labels) — Distributed state machine; "Markdown as a Database"
3. **The Brain** (`src/sentinel/orchestrator_sentinel.py`) — Async polling background service; claims tasks; dispatches via shell bridge
4. **The Hands** (opencode Worker DevContainer) — Isolated code execution environment

## Environment Setup

**Always activate the virtual environment before running commands:**
```bash
# Dependencies are managed via uv
uv sync --dev           # Install all including dev tools
# OR use the venv Python directly:
.venv/bin/python3 ...
```

**Required environment variables** (copy `.env.example` → `.env`):
- `GITHUB_TOKEN` — GitHub PAT with `repo` + `project` scopes
- `ZHIPU_API_KEY` — ZhipuAI GLM model access
- `WEBHOOK_SECRET` — HMAC secret for GitHub webhook validation

## Build / Run Commands

```bash
# Install dependencies
uv sync --dev

# Run tests (use .venv/bin/python3 to avoid uv run hanging on some environments)
.venv/bin/python3 -m pytest tests/ -v --no-cov

# Run tests with coverage
.venv/bin/python3 -m pytest tests/ --cov=src --cov-report=term-missing

# Lint
.venv/bin/ruff check src/ tests/

# Format check
.venv/bin/black --check src/ tests/

# Start Notifier service locally
.venv/bin/uvicorn src.notifier.notifier_service:app --host 0.0.0.0 --port 8000

# Start Sentinel service locally
.venv/bin/python3 -m src.sentinel.orchestrator_sentinel

# Start all services with Docker Compose
docker compose up

# Build Docker images
docker build -f Dockerfile.sentinel -t sentinel .
docker build -f Dockerfile.notifier -t notifier .
```

**IMPORTANT:** When running pytest, always use `.venv/bin/python3 -m pytest` directly.
`uv run pytest` can hang in some terminal environments when piping output.
The `--no-cov` flag is useful for fast runs; omit it to get coverage reporting.

## CI/CD Pipelines

- **`validate.yml`** — runs on PRs: lint (ruff), type-check (mypy), tests (pytest), security scan
- **`publish-docker.yml`** — builds Dockerfile, pushes to `ghcr.io/intel-agency/workflow-orchestration-queue-uniform39` with `main-latest` and versioned tags
- **`prebuild-devcontainer.yml`** — triggered by `publish-docker` completion; builds and caches prebuilt devcontainer image
- **`orchestrator-agent.yml`** — triggered by GitHub events (issues, PR comments); assembles prompt, spins up devcontainer, runs `opencode --agent Orchestrator`

## Code Conventions

- Python 3.12+: use `StrEnum` (not `str, Enum`), `from __future__ import annotations`
- `uv` is the only package manager — never use `pip install` directly in project workflows
- All Python modules include a module-level docstring
- Pydantic v2 models for all data structures crossing service boundaries
- HMAC validation must occur BEFORE JSON parsing in webhook handlers
- Shell bridge interactions via `./scripts/devcontainer-opencode.sh` exclusively (ADR-07)
- Test files follow `test_<module>.py` naming in `tests/`

## GitHub Label State Machine

```
agent:queued → agent:in-progress → agent:success
                     ↓
              agent:reconciling
                     ↓
         agent:error / agent:infra-failure / agent:impl-error / agent:stalled-budget
```
